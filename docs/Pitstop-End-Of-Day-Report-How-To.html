Pitstop end-of-day report — how to

End-of-event report for Pitstop terminal sales and outside merch / raffle only. ClubPOS bar tabs are not included.

Where: Reports → Pitstop end-of-day report
Top bar: Cash to deposit, Net profit, Save and Export
Jump chips: Summary · Event · Terminal · Square · Outside · Prizes · Costs

1. Normal run (Square working)

Event

  1. Set Event name.
  2. Confirm Report date (event day).
  3. Check staff name looks correct.

Terminal (automatic)

  • Terminal cash and Terminal card come from Pitstop POS for that date.
  • No typing needed in this section.

Square

  1. Tap Refresh Square.
  2. Check POS terminal (Square Terminal 0070) — transaction count and amount.
  3. Check Outside terminal (Flounderers02) — transaction count and amount.
  4. Tap either amount to view the transaction list if you need to verify.
  5. Confirm Combined Square gross looks like the day’s total card sales.

Outside sales (paper sheet)

  1. Enter Cash qty and Cash $ from the paper sheet (Merch and Raffle).
  2. Card quantities / totals should fill from Flounderers02 when Square loaded correctly.
  3. Optional: Reload lines, or hide unused merch lines.

Prizes, costs, float, cash count

  1. Enter prize giveaway quantities (these deduct stock when archived).
  2. Add any expenses.
  3. Enter Inside float and Outside float.
  4. Enter Cash counted and Float removed if you did a physical count.
  5. Add archive notes if useful for later.

Finish

  1. Check top totals and the right-hand Breakdown.
  2. Tap Save and Export (PDF saves / opens).
  3. If prompted, archive the Pitstop period (Admin / Treasurer) so sales are locked and stock is deducted.

2. Manual Square card fallback

Use this when the outside terminal does not show up, or Square import fails / looks wrong.

When to use: Outside terminal shows 0 txns / $0 but Flounderers02 took card, or Square failed for the whole day.

Enter the combined Square card total for the event day (POS terminal + outside terminal), not outside alone.
  1. Go to Square.
  2. Tap Show manual card fallback.
  3. From Square (app / dashboard / close-out), get the total card gross for the event day.
  4. Tap Total Square card gross (manual) and enter that total.
  5. Check Outside card (derived) = your total − Pitstop terminal card.
  6. Confirm status shows manual fallback is active.
  7. On Outside sales, enter Card qty (and Card $ if shown) from the paper sheet — stock needs this when Square product lines did not load.
  8. Tap Save and Export as usual.
  9. Use Clear later if you want to remove the override and return to automatic Square totals.

Example

Square close-out total card = $1,200
Pitstop terminal card = $800
Outside card used in report = $400

3. What is counted (and what is not)

  • Counted from ClubPOS: Pitstop terminal cash and card only. Bar tabs / bar sales are never included.
  • Counted from Square: Card payments for the report date at the configured Square location. Payments that match a Pitstop sale = POS terminal. Payments that do not match ClubPOS = outside terminal (Flounderers02 / merch table).
  • Important: If the bar also takes Square card on the same Square location, those bar card payments can show up as “outside” because they are not linked to a Pitstop sale. Check the outside transaction list if outside totals look too high.
  • Not counted: Bar tab cash, bar tab product lines, and anything outside the report date.

4. Mismatch / red banner

  1. Tap the Square amounts and review the transaction lists.
  2. Confirm the report date, then try Refresh Square.
  3. If Square is still wrong, use the manual card fallback above.
  4. First Save and Export may warn you — tap again to save anyway once you have checked the figures.

5. End of event checklist

6. Common mistakes

  • Wrong date — Square and POS totals will not match the event.
  • Entering only outside card into manual fallback — enter the combined Square card total for the day.
  • Using fallback but skipping card qty on outside lines — money can look right but stock will be wrong.
  • Expecting bar tabs from ClubPOS — they are never included.
  • Bar Square on the same location — those card payments can inflate outside totals; check the outside txn list.
